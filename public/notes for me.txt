user runs DSQ Enterprise.exe
it downloads launcher.exe after checking launcher.version and verifying version, and runs it.
launcher.exe runs main.js from local folder.
that runs index.html inside it.
inside the index.html window
  button submit - runs sub.exe from local folder C/DSQ Enterprise/apps/sub.exe
  button updates - runs downloads and runs c/DSQ..../updates/updater.exe


updater.exe:
  checks updates.txt from vercel server 
  the updater.exe then downloads those file names and overwrites on top of existing file where ever in subfolder or parent in c/DSQ Enterprise.

notes for updates.txt
 ✅ updates.txt example
"
Download main.js > root/main.js
Download app/abc.exe > root/FolderA/abc.exe
Delete root/OldFolder/oldfile.txt
Rename root/FolderB/temp.txt > final.txt
"

Download → fetches from https://dsq-beta.vercel.app/<path> and saves locally.

Delete / Rename → only affects local C:\DSQ Enterprise files.
